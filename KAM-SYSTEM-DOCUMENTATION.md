# KAM System Documentation
## Key Account Management Application

---

## Project Overview

| Field | Details |
|-------|---------|
| **Project Name** | KAM (Key Account Management) - Approved UI |
| **Project Type** | Enterprise Web Application (Next.js 14) |
| **Primary Goal** | Manage sales inquiries, quotations, approvals, projects, and client relationships for a packaging/printing business |
| **Tech Stack** | Next.js, React, TypeScript, Material React Table, Tailwind CSS, Shadcn UI |
| **API Backend** | api.indusanalytics.co.in |

---

## User Roles

| Role | Permissions |
|------|-------------|
| **KAM (Key Account Manager)** | Create inquiries, Create quotations, View own data |
| **HOD (Head of Department)** | View team data, L1 Approvals (Margin â‰¥ 5%), Reports |
| **Vertical Head** | View all data, L2 Approvals (Margin < 5%), Strategic decisions |
| **Admin** | Full system access |

---

## Core Modules

1. **Dashboard** - KPIs, Charts, Overview metrics
2. **Inquiries Management** - Create, track, convert inquiries
3. **Quotations Management** - Costing, pricing, margin calculation
4. **Approvals Workflow** - L1/L2 approval routing
5. **Projects** - SDO, JDO, Commercial, PN tracking
6. **Clients Management** - Customer master data
7. **Rate Queries** - Rate request management
8. **Chat/AI Assistant** - AI-powered assistance

---

## Key Flows

```
Inquiry â†’ Quotation â†’ Approval â†’ SDO â†’ JDO â†’ Commercial â†’ PN â†’ Delivery
```

## Data Inputs & Outputs

### Inputs
- Customer inquiries (manual & email scraper)
- Quotation details (costs, margins, specifications)
- Approval decisions
- Project status updates

### Outputs
- Quotation documents
- Approval status reports
- Project tracking data
- Customer history

## Integrations

- Backend API (api.indusanalytics.co.in)
- Email scraper for inquiries
- AI Chat assistant
